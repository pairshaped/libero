# Todos example

A full CRUD todo list app showing how to build a stateful backend with ETS persistence and a Lustre client that communicates over RPC.

## What's in here

- `src/todos.gleam` - server entry point (generated by libero, serves static files and handles RPC)
- `src/server/handler.gleam` - handler functions for get, create, toggle, and delete (each is an RPC endpoint)
- `src/server/handler_context.gleam` - server context with in-memory todo storage
- `shared/src/shared/types.gleam` - domain types: Todo, TodoParams, TodoError
- `clients/web/src/app.gleam` - Lustre SPA with a form, todo list, and error handling

## How it works

Each public function in `handler.gleam` becomes an RPC endpoint. The scanner detects them by their signature (takes `HandlerContext`, returns `#(Result(value, error), HandlerContext)`, uses only shared types). Codegen generates a `ClientMsg` variant per function and typed client stubs.

The client calls `rpc.create_todo(params: TodoParams(title: "Buy milk"), on_response: GotCreated)`. The server handler processes the call and returns a typed `Result`. The client receives it as `RemoteData(Todo, TodoError)` for loading/success/failure states.

No SSR or shared views in this example. The client builds the entire UI in JavaScript.

## Running it

```sh
gleam run -m libero -- build
gleam run
```

Open http://localhost:8080 in your browser.

## Running tests

```sh
gleam test
```

Tests cover CRUD operations, empty title validation, and error handling for nonexistent records.
