# Todos example

A full CRUD todo list app showing how to build a stateful backend with ETS persistence and a Lustre client that communicates over RPC.

## What's in here

- `src/todos.gleam` - server entry point (generated by libero, serves static files and handles RPC)
- `src/server/handler.gleam` - handles Create, Toggle, Delete, and LoadAll messages
- `src/ets_store.gleam` - ETS-backed storage with auto-incrementing IDs
- `shared/src/shared/messages.gleam` - message types, Todo record, and error types (NotFound, TitleRequired)
- `clients/web/src/app.gleam` - Lustre SPA with a form, todo list, and error handling

## How it works

The client sends typed RPC messages to the server (e.g. `Create(TodoParams(title: "Buy milk"))`). The server handler processes each message, reads/writes ETS, and returns a typed response. The client updates its UI based on the response using `RpcData` for loading/success/failure states.

No SSR or shared views in this example. The client builds the entire UI in JavaScript.

## Running it

```sh
gleam run -m libero -- build
gleam run
```

Open http://localhost:8080 in your browser.

## Running tests

```sh
gleam test
```

Tests cover CRUD operations, empty title validation, and error handling for nonexistent records.
