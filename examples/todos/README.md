# Todos example

A full CRUD todo list app showing how to build a stateful backend with in-memory storage and a Lustre client that communicates over RPC.

## What's in here

- `server/src/todos.gleam` - server entry point (generated by libero, serves static files and handles RPC)
- `server/src/handler.gleam` - handler functions for get, create, toggle, and delete (each is an RPC endpoint)
- `server/src/handler_context.gleam` - server context with in-memory todo storage
- `shared/src/shared/types.gleam` - domain types: Todo, TodoParams, TodoError
- `clients/web/src/app.gleam` - Lustre SPA with a form, todo list, and error handling

## How it works

Each public function in `handler.gleam` becomes an RPC endpoint. The scanner detects them by their signature (takes `HandlerContext`, returns `#(Result(value, error), HandlerContext)`, uses only shared types). Codegen generates a `ClientMsg` variant per function and typed client stubs.

The client calls `rpc.create_todo(params: TodoParams(title: "Buy milk"), on_response: GotCreated)`. The server handler processes the call and returns a typed `Result`. The client receives it as `RemoteData(Todo, TodoError)` for loading/success/failure states.

No SSR or shared views in this example. The client builds the entire UI in JavaScript.

## Running it

```sh
bin/dev
```

Open http://localhost:8080 in your browser.

## Running tests

```sh
bin/test
```

Tests cover CRUD operations, empty title validation, and error handling for nonexistent records.

## Layout

```
todos/
├── server/         backend (Erlang), runs the libero RPC server
├── shared/         cross-target types
├── clients/web/    Lustre SPA (JavaScript)
└── bin/            dev and test entry points
```

## Adding a handler

Add a function in `server/src/handler.gleam`. It must:
- Be `pub`
- Take `HandlerContext` as its last parameter (named `state`)
- Return `#(Result(value, error), HandlerContext)`
- Use only types from `shared/` or builtins

Then run `bin/dev` (or `cd server && gleam run -m libero`) to regenerate dispatch + client stubs.
